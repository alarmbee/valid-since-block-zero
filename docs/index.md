- Strukturált esetleírások (kérdés → tényállás → vonatkozó források → nyitott kérdések)

- Anonimizált levelezések és válaszok az érintett hatóságoktól és szolgáltatóktól

## Hogyan használd?

### Ha módosítást szeretnél beküldeni

Nem csak **kérdést** lehet beküldeni: bármelyik oldalt szerkesztheted, ahol megjelenik az **„Edit with GitHub”** (Szerkesztés GitHubon) link.
Az online szerkesztéshez **GitHub fiók szükséges**.

Módosítás (vagy pontosítás) beküldésének legegyszerűbb módja egy pull request (PR) készítése.

Két kényelmes út van:

1) **Az oldalról („Edit with GitHub”)**

- Nyisd meg a megfelelő oldalt a weben (például a **Kérdések** listát, egy sablont, egy esetet, vagy bármelyik másik oldalt).
- Kattints az oldal „Edit with GitHub” (Szerkesztés GitHubon) linkjére.
- A GitHub megnyitja a fájlt szerkesztésre (ehhez be kell jelentkezned a GitHub fiókoddal). Ha nincs közvetlen jogosultságod a repository-hoz, a GitHub felajánlja, hogy készít egy forkot.
- Végezd el a módosítást, majd a GitHub felületén mentsd el („Commit changes”).
- A mentés után a GitHub felajánlja a pull request létrehozását („Create pull request”). Ezt elküldve tudjuk átnézni és beemelni.

2) **Közvetlenül a GitHubon**

- Nyisd meg a repository-t a GitHubon.
- Navigálj a megfelelő mappába (például `docs/questions/`, `docs/templates/`, `docs/cases/`, `docs/conclusions/`).
- Meglévő oldalnál kattints a fájl szerkesztésére (ceruza ikon), új oldalnál használd az „Add file” lehetőséget.
- A végén készíts pull requestet a módosításokra.

Tipp: új oldalhoz érdemes a meglévő fájlokat mintának venni (például `Q-002.md`).

### Ha nem így szeretnéd

Ha nem szeretnél pull requestet nyitni, írj a lap alján található **komment szekcióban**, vagy vedd fel velünk a kapcsolatot e-mailben (lásd: [Impresszum](/impresszum)).

### Ha esetet publikálnál

Eset (valós levelezés/válasz) publikálása előtt **mindenképpen vedd fel velünk a kapcsolatot** (lásd: [Impresszum](/impresszum)).
Az eseteket csak anonimizált formában tesszük közzé, és segítünk abban, hogy semmilyen személyes adat ne kerüljön ki.

### Email sablonok használata és publikálása

- Az email sablonokat **szabadon felhasználhatod** az adott szervek felé: letölthetők, másolhatók, és a saját esetedre szabhatók.
- Ha új sablont publikálnál, ugyanúgy jöhet:
  - „Edit with GitHub” → módosítás → commit → pull request, vagy
  - közvetlen GitHub szerkesztés → pull request, vagy
  - e-mailben elküldöd nekünk (lásd: [Impresszum](/impresszum)).
